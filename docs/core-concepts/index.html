Core Concepts

Understanding these concepts will help you get the best results from Img2Num.

Color Spaces​

Img2Num supports two color spaces for k-means clustering:

SpaceIDDescriptionUse when…
CIE LAB0Perceptually uniform — distances match human color perception.Accurate color matching matters more than speed.
sRGB1Faster computation in the native display space.You need speed and color accuracy is secondary.

Bilateral Filtering​

The bilateral filter applies two Gaussian kernels:

Spatial Kernel​

Typical value: 3

σs\sigma_{s}σs​ (sigma_spatial) is the parameter that controls neighbor pixel smoothing in the image plane. Neighbors are pixels that are close to each other.

Above, σr = 20 for RGB and σr = 15 for CIELAB.

Range Kernel​

Typical value: 50

σr\sigma_{r}σr​ (sigma_range) is the parameter that controls smoothing of pixels with similar intensity values.

For example, a pixel with value rgba(213,13,67) and another with rgba(195,17,87) may be considered similar and get smoothed together.

K-Means Clustering​

K-means groups pixels into k clusters based on color distance in the chosen color space.

  • k (num_colors): How many colors the output should contain.
  • max_iter: Maximum iterations allowed before forcing termination.
note

k and max_iter are not guarantees.

  • k cannot force new colors: if the image only has 2 colors and k=5k=5k=5, the image cannot gain more color.
  • max_iter is an upper limit only: if maxiter=999max_{iter}=999maxiter​=999 and it only takes 50 iterations to cluster the image, the function will return early (before the 999th iteration).
Choosing the Value of k

Larger images benefit from more colors (k), but too many will produce noisy contours.
